一、OMNeT简介

OMNeT++(Objective Modular Network TestBed in C++) 是开源的基于组件的模块化的开放网络仿真平台，是近年来在科学和工业领域里逐渐流行的一种优秀的网络仿真平台。

OMNeT++作为离散事件仿真器，具备强大完善的图形界面接口和可嵌入式仿真内核，同NS2等仿真平台相比，OMNeT++可运行于多个操作系统平台，可以简便定义网 络拓扑结构，具备编程，调试和跟踪支持等功能。

OMNeT++ 主要用于通信网络和分布式系统的仿真，目前最高版本为5.7。

二、配置仿真环境

2.1 实验环境

  • Ubuntu 18.04
  • OMNeT 5.5.1
  • INET Framework 4.1.2 (负责仿真多种以太网协议)
  • Nesting （负责仿真多种TSN模型）

    注：以上版本已由Nesting官方完成稳定性验证，建议安装此版本，防止出错。

2.3 常见问题

  • 安装过程中不要使用root用户，会导致文件无法编译
  • 大多数教程中的环境变量设置与自己本机的环境变量不同，建议进入OMNeT文件夹后采用pwd指令查看本地具体路径
  • 配置过程中需要下载大量国外资源，建议配置apt代理和git代理

    三、OMNeT++框架

3.1 OMNeT++组成

OMNeT++主要由六个部分组成：

  • 仿真内核库 （simulation kernel library, Sim） ：最重要的部分，用户编写的仿真程序要与Sim链接

  • 网络描述语言的编译器 （network description compiler, Nedc） ：网络拓扑文件编译器

  • 图形化的网络编辑器（graphical network description editor, Gned）：可用于编写简单的网络拓扑，复杂拓扑需要直接编写ned文件

  • 仿真程序的图形化用户接口 （Tkenv） ：通过图形化界面仿真

  • 仿真程序的命令行用户接口 （Cmdenv） ：通过命令行界面仿真，支持批处理

  • 图形化的输出工具 Plove 和 Scalar ：输出仿真结果，用于分析

3.2 OMNeT++文件简介

OMNeT++中涉及多个文件，其中我们重点关注.ned .cc .ini .vec .sca文件。

  • .ned文件：（network description file）负责描述网络拓扑结构
  • .cc文件 ： 负责描述网络中的
  • .ini文件： 负责描述网络组件配置属性，如MAC地址，数据报文大小等
  • .vec文件： 统计网络流量，生成折线图
  • .sca文件： 统计网络流量，生成柱状图
